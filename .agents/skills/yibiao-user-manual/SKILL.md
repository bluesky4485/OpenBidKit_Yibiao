# 易标用户手册

独立处理“使用说明”和“v2 版本更新日志”两类任务。只执行用户选定的任务，不因版本升级自动改写使用说明，也不因更新使用说明自动同步版本日志。

## 开始前

1. 完整阅读 [project-contract.md](references/project-contract.md)。
2. 判断任务类型：
   - 用户明确要求创建、更新、检查使用说明或重采截图时，执行“使用说明任务”。
   - 用户明确要求更新版本日志时，执行“版本日志任务”。
   - 用户明确要求两项都做时，分别建立目标范围并依次执行。
   - 用户没有说明时，先询问要“更新使用说明”“更新 v2 版本日志”还是“两项都执行”，得到选择后再继续。
3. 从当前工作目录向上查找 `AGENTS.md` 并遵守；进入客户端代码前阅读 `client/开发说明.md`。
4. 检查工作区状态，保留用户已有修改，不修改 `client/`、`analytics/` 或所选任务范围外内容。
5. 对范围外文件记录修改前哈希，完成后复核哈希未变化。

## 使用说明任务

基于项目代码和发行版真实界面制作手册。以用户可见的发行版为最终依据，不根据未发布代码臆造功能。

### 确认范围和发行版

1. 建立目标清单：目标 Markdown、原图、标注图。用户指定功能时只处理这些文件；未指定时才使用完整手册范围。`v2版本更新日志.md` 不属于本任务范围。
2. 按以下顺序确认可执行文件：
   - 使用用户本次明确提供的有效路径。
   - 否则使用项目参考中的默认路径，但必须确认文件存在且产品名称为“易标投标工具箱”。
   - 默认路径失效、路径未知或出现多个候选时，立即询问用户提供发行版路径并暂停；不得运行 Vite/Electron 开发版代替。
3. 需要控制 Windows 界面时，使用可用的 Windows 应用控制能力并遵守其说明。只导航、展开、切换页签或点击“下一步”查看已有结果，不点击会执行任务的按钮。

### 检查数据门槛

- 配置页只需界面可见，不要求业务结果。
- 流程页必须已有与目标章节匹配的导入文件、解析结果、目录、正文、知识库文档或检查结果。
- 只有空白页、零条记录、占位内容或未执行状态时，视为没有数据。

没有数据时停止文档制作，不创建推测性说明，也不自行运行流程。说明缺少哪个功能的数据、需要用户在发行版中完成哪些步骤，并提示完成后再次调用本 Skill。

### 创建或更新文档

1. 对照 `menuConfig.ts`、`AppRouter.tsx`、目标 `features/` 页面和发行版界面，确认功能已完成且普通用户可稳定使用。
2. 跳过“正在开发中”、禁用、开发者模式专用或只存在于代码而发行版不可见的功能。
3. 手册不存在时创建标准目录；手册存在时保留未变化章节和截图。
4. 使用简单中文说明“在哪里点、等待什么、如何确认完成、最后得到什么”。界面按钮和菜单名称保持原文。
5. 用户只要求编写或更新某部分时，只编辑目标 Markdown 和目标截图；范围外问题仅报告，不修复。
6. 不创建根 README、总目录、上一篇、下一篇、返回链接或其他章节导航。

### 采集和标注截图

1. 最大化发行版窗口，等待动画和提示消失，避免鼠标遮挡关键内容。
2. 使用 [capture-window.ps1](scripts/capture-window.ps1) 按物理像素窗口边界保存无损 PNG 到 `使用说明/images/`。
3. 目视检查原图清晰、页面状态正确、API Key 为掩码且没有不应公开的信息。
4. 阅读 [annotation-format.md](references/annotation-format.md)，为每张图准备 1–5 个短标注的 JSON 坐标。
5. 使用 [annotate-screenshot.ps1](scripts/annotate-screenshot.ps1) 生成 `使用说明/images/标注/` 下的同名 PNG。最终标注必须由该确定性脚本叠加，不使用生成式图片编辑重绘界面。
6. 目视检查标签不遮挡关键文字、箭头准确落到按钮或需复核区域，且标注图尺寸与原图一致。

## 版本日志任务

1. 完整阅读 [changelog-standard.md](references/changelog-standard.md)，只处理 `使用说明/v2版本更新日志.md`。
2. 确认仓库不是缺少历史的浅克隆，并确认日志最新版本存在对应的正式 `v2.*` Git 标签。历史或比较基线缺失时停止并报告，不猜测内容。
3. 日志不存在时先询问用户从哪个 v2 版本开始整理，不擅自回溯全部 v2 历史。
4. 更新前记录日志第一条版本作为校验基线，再找出其后尚未记录的正式 `v2.*` 标签。没有新标签时保持文件不变并报告“没有新的正式版本”。
5. 按版本逐一检查提交记录、版本差异、实际代码和用户可见文案。把提交标题只当线索，合并重复变化并排除重构、测试、CI、依赖整理等用户不可感知内容。
6. 按规范写成专业软件更新日志，只在一级标题后插入缺失版本，整体保持版本倒序，不改写既有版本条目。
7. 不启动发行版、不采集截图，也不修改 `配置/`、`使用/` 或 `images/` 下的内容。

## 验证和交付

1. 使用说明任务：使用 [validate-manual.ps1](scripts/validate-manual.ps1) 校验目标范围；未限定范围时校验完整使用说明，不启用版本日志校验。
2. 版本日志任务：使用同一脚本的 `-ValidateChangelog -Scope 'v2版本更新日志.md' -ChangelogBaselineVersion '<更新前日志最新版本>'` 校验新增版本；新建日志时省略基线参数。
3. 两项都执行时分别完成上述校验，避免一项的范围问题阻断另一项。
4. 复核范围外文件哈希、工作区差异和目标引用，确认没有越界修改。
5. 如果使用说明任务由 Skill 启动软件，完成后关闭；如果软件原本已由用户打开，则保持打开并说明当前停留页面。
6. 最终按所选任务汇报：
   - 使用说明：处理范围、创建或更新的章节、截图数量、验证结果和需要用户补充的数据。
   - 版本日志：新增的版本号和条目分类；没有新标签时明确说明“没有新的正式版本”。

## 常用调用

- `$yibiao-user-manual`：询问本次执行哪类任务。
- `$yibiao-user-manual 更新使用说明`：创建或更新完整使用说明。
- `$yibiao-user-manual 只更新废标项检查文档`：只处理指定功能。
- `$yibiao-user-manual 更新 v2 版本日志`：只增量更新正式版本日志。
- `$yibiao-user-manual 更新使用说明和 v2 版本日志`：分别执行两项任务。
- `$yibiao-user-manual 仅检查使用说明，不修改`：执行使用说明只读检查。
